# Raven 350 Classic Information System

## Overview

Raven 350 Classic Information System adalah sebuah **single-product information & decision support system** yang dirancang untuk membantu pengguna memahami **kecocokan sebuah motor klasik** berdasarkan kebutuhan dan preferensi penggunaan.

Proyek ini tidak berfokus pada transaksi atau penjualan, melainkan pada **penyajian informasi yang terstruktur, jujur, dan berbasis pertimbangan**, sehingga pengguna dapat mengambil keputusan dengan lebih sadar.

---

## Problem Background

Banyak situs produk hanya menonjolkan visual dan spesifikasi tanpa membantu pengguna memahami:

* apakah produk tersebut cocok untuk kebutuhan mereka
* konsekuensi penggunaan jangka panjang
* kompromi yang harus diterima

Hal ini sering menyebabkan keputusan impulsif yang tidak sesuai dengan konteks penggunaan.

Raven 350 Classic Information System dibuat untuk menjawab masalah tersebut melalui pendekatan **Sistem Informasi**.

---

## System Objective

Tujuan utama sistem ini adalah:

* Menyediakan informasi produk yang **ringkas, relevan, dan mudah dipahami**
* Membantu pengguna mengevaluasi kecocokan produk berdasarkan preferensi pribadi
* Menyajikan hasil evaluasi dalam bentuk **kesimpulan sistem**, bukan opini

---

## System Scope

Sistem ini memiliki ruang lingkup:

* Satu produk (single product system)
* Pengguna umum (publik)
* Tidak menggunakan database
* Tidak melakukan transaksi

Fokus utama sistem adalah **information clarity dan decision support**.

---

## Features

### 1. Product Information Presentation

* Ringkasan produk
* Karakter utama
* Fakta singkat dan spesifikasi ringan
* Kelebihan dan risiko penggunaan

### 2. Interactive Product Carousel

* Menampilkan beberapa sudut pandang produk
* Animasi otomatis dengan interaksi yang ramah pengguna

### 3. User Preference Input

Pengguna dapat memasukkan:

* Tujuan penggunaan (harian / hobi)
* Budget perawatan bulanan

### 4. Rule-Based Decision Support

Sistem menggunakan pendekatan **rule-based evaluation** untuk:

* Mengolah preferensi pengguna
* Membandingkannya dengan karakter produk
* Menghasilkan kesimpulan kecocokan

### 5. Visual System Feedback

* Status hasil (Cocok / Perlu Pertimbangan / Kurang Disarankan)
* Penjelasan berbasis alasan sistem

---

## Decision Logic (Sistem Informasi Aspect)

Sistem ini **bukan AI** dan **bukan machine learning**.

Pendekatan yang digunakan adalah:

* Rule-based Decision Support System
* Aturan diturunkan dari karakteristik produk
* Setiap input pengguna dievaluasi secara transparan

Contoh logika:

* Jika penggunaan harian tinggi → skor dikurangi
* Jika orientasi hobi sesuai → skor bertambah
* Jika budget tidak mencukupi → sistem memberikan peringatan

Pendekatan ini membuat sistem:

* Mudah dijelaskan
* Mudah dikembangkan
* Relevan dengan konsep Sistem Informasi
